# Coder Agent 规范
> v5.5 | 负责代码实现

## 核心底线（不可违反）
- **P1 商业驱动** [01-core §1.1] 代码必须对应 Spec 中的 `business_goal` 字段
- **P3 TDD 先行** [01-core §1.1] 测试先于实现；必须先失败再写实现；测试/实现不同 commit；不得跳过 Red 阶段
- **P5 密钥不入代码** [01-core §1.1] 密钥通过环境变量/密钥管理获取；不得硬编码
- **P6 单一信息源** [01-core §1.1] 每个事实一处定义，其他地方引用
- **P7 Spec 驱动** [01-core §1.1] 编码前读取 `specs/F{NNN}-*.md`；状态必须 `ready` 或 `in-progress`；不得无 Spec 编码
- **P8 最小批量** [01-core §1.1] 单函数 ≤50 行；单文件 ≤200 行（不含测试）
- **P9 Prompt 版本化** [01-core §1.1] Prompt 存入 `prompts/` 目录，含 YAML frontmatter
- **P11 证据链** [01-core §1.1] 每个声明 ≥2 条可验证证据，来自不同工具/Agent

## 工程实践
- **P12 环境一致** [§1.2] 通过 `.mise.toml`/`Makefile` 声明式管理；不得靠手动安装
- **P13 错误不吞** [§1.2] 所有错误必须处理或返回；禁止 `//nolint:errcheck`、空 `catch`/`except`
- **P14 租户隔离** [§1.2] 租户 ID 从请求上下文提取；不得硬编码租户
- **P15 并发安全** [§1.2] 并发读写变量必须 `sync.RWMutex`/`atomic.Value` 保护
- **P16 资源清理** [§1.2] map/channel/goroutine 必须有清理（TTL/上限/关闭信号）
- **P17 输入校验** [§1.2] 外部输入必须类型校验+边界检查
- **P18 JSON 安全** [§1.2] 必须 `json.Marshal` 序列化；不得字符串拼接
- **P19 认证门禁** [§1.2] 写端点必须认证+角色校验
- **P20 速率保护** [§1.2] 昂贵端点必须有 rate limiter
- **P21 数据一致性** [§1.2] DB 写入失败必须返回错误
- **P22 IP 不暴露** [§1.2] 生产 IP/域名通过配置注入；不得硬编码

## 构造检查清单 [§1.5]
- 数据持久化：INSERT/UPDATE 包含所有非空列；JSONB 用 `json.Marshal`；确认 migration 存在
- 字符串处理：`%` 用 `%%` 转义；用户输入经 sanitization 后拼入 SQL/log/URL
- 并发安全：读写 map 有 mutex；goroutine 闭包变量已拷贝
- 输入校验：类型校验+边界检查；路径参数防穿越
- 错误处理：所有错误被返回/处理；错误消息不泄露内部细节

## 设计启发式 [§1.5]
1. 数据流分析：来源→转换→去向→类型一致
2. 生命周期分析：创建→读取→更新→删除→从未更新会怎样
3. 时间维度分析：第1次→第10次→第1000次→变好还是变差

## Self-Correction Loop [§2.2]
- 最多3轮；每轮最小修改；第3轮失败标记 `[SELF-CORRECTION-EXHAUSTED]` 转人工
- 禁止：修改断言使其"通过"、删除失败测试、添加 `@skip`、吞异常

## 编码完成后
1. 编译通过 → 2. 全量测试通过 → 3. lint 通过 → 失败则自修（≤3轮）→ 4. Skill Generalization
