# AI Coding 规范 v5.2：安全优先的 Auto-Coding

> 版本：v5.2 | 2026-04-17
> 适用：1-3 人小团队 / auto-coding 模式
> 核心原则：v4 的安全边界不动摇，auto-coding 在安全边界内最大化自主性，P23 方案设计驱动编码全流程

---

## 自治等级模型

本规范定义 4 个自治等级，**每个等级继承上一级的约束**：

| 等级 | 名称 | 人类干预点 | 适用场景 |
|------|------|-----------|---------|
| **L1** | 辅助编码 | 每一步 | 新手团队、安全敏感项目 |
| **L2** | 半自主编码 | 每个 PR 合并前 | 日常开发（推荐默认等级） |
| **L3** | 受限自主编码 | 每个 PR 合并前 + DCP 门禁 | 夜间/周末开发、自修复 CI |
| **L4** | 完全自主编码 | DCP 门禁 + 定期审计 | 成熟团队、低风险变更 |

### 约束矩阵

| 约束 | L1 | L2 | L3 | L4 |
|------|----|----|----|----|
| **TDD 先行（P3）** | 强制 Red→Green | 强制 Red→Green | 强制 Red→Green | 强制 Red→Green |
| **人工审查（P4）** | 每次提交前 | 每个 PR 合并前 | 每个 PR 合并前 | 定期审计（每周） |
| **DCP 门禁（P2）** | 强制 | 强制 | 强制 | 简化 DCP |
| **Spec 驱动（P7）** | 强制 | 强制 | 强制 | 可简化为 backlog item |
| **最小批量（P8）** | 强制 | 强制 | 强制 | 强制 |
| **Prompt 版本化（P9）** | 强制 | 强制 | 强制 | 自动持久化 |
| **数据分级（P10）** | 强制 | 强制 | 强制 | 强制 |
| **P23** | **强制** | **强制** | **强制** | **强制** |
| **幻觉检测（新增）** | 强制 | 强制 | 强制 | 强制 |
| **自修复限制** | 最多 3 轮 | 最多 3 轮 | 最多 3 轮 | 最多 3 轮 |
| **MCP 访问** | 只读 | 读写（过滤） | 读写（过滤+脱敏） | 读写（过滤+脱敏+审计） |
| **自动合并** | 禁止 | 禁止 | 禁止 | 仅限 trivial fix（lint 修复） |

### 升级路径

```
L1 → L2：团队积累 20 个 PR 无安全事故
L2 → L3：L2 运行 1 个月，自主成功率 > 70%
L3 → L4：L3 运行 3 个月，自主成功率 > 85%，审计通过率 > 95%
```
