## 第 1 章：数据库迁移

### 1.1 TDD 迁移

数据库变更必须先写测试：
1. 测试当前 schema 状态
2. 测试迁移脚本执行
3. 测试回滚脚本（如果有）
4. 测试迁移后 schema 符合预期

### 1.2 Expand-Contract 模式

| 步骤 | 操作 | 说明 |
|------|------|------|
| **Expand** | 新增列/表，允许 NULL | 不破坏现有读写 |
| **双写** | 新旧路径同时写入 | 确保数据一致 |
| **迁移** | 后台迁移历史数据 | 分批、可暂停 |
| **切换** | 读取切换到新路径 | 灰度、可回滚 |
| **Contract** | 删除旧列/表 | 确认无引用后执行 |

### 1.3 迁移 Gate

- 在干净 DB 上执行迁移 + 回滚 + destructive change 检测
- 迁移脚本必须幂等
- 禁止直接修改生产数据（必须通过迁移脚本）
- 回滚脚本必须测试通过

## 第 2 章：API 契约

### 2.1 Spec 驱动的 API 设计

API 端点必须在 Spec 中定义：
- 路径、方法、请求参数（类型、必填、默认值）
- 响应格式（成功/错误）
- 认证要求
- 速率限制

### 2.2 API 漂移检测

CI 检查实现是否偏离 Spec 中的 API 定义：
- 新增端点未在 Spec 中声明 → 警告
- Spec 中声明的端点未实现 → 阻断
- 响应格式与 Spec 不一致 → 阻断

### 2.3 向后兼容

| 变更类型 | 兼容性 | 要求 |
|---------|--------|------|
| 新增可选参数 | 兼容 | 无需版本号变更 |
| 新增端点 | 兼容 | 无需版本号变更 |
| 删除端点 | 不兼容 | 需废弃周期 + 版本号 |
| 修改参数类型 | 不兼容 | 需版本号变更 |
| 新增必填参数 | 不兼容 | 需版本号变更 |

## 第 3 章：性能基线

### 3.1 性能预算

每个端点必须有性能预算：
- P50 响应时间
- P99 响应时间
- 最大并发数
- 内存上限

### 3.2 CI 性能检查

L4 层执行：
- 基准测试 vs 上次基线
- 回归 >5% → 警告
- 回归 >10% → 阻断
- 不得由 AI 自修性能退化（需人工分析）

### 3.3 N+1 检测

ORM 查询必须检测 N+1 问题：
- CI 中开启查询日志
- 单次请求的 SQL 查询数设上限
- 慢查询（>100ms）必须记录原因

## 第 4 章：部署与回滚

### 4.1 部署策略

| 策略 | 适用场景 | 说明 |
|------|---------|------|
| 金丝雀发布 | 默认 | 5%→25%→50%→100%，每阶段验证指标 |
| 蓝绿部署 | 零停机要求 | 双环境切换，失败立即切回 |
| 滚动升级 | 大规模集群 | 逐批替换实例 |

### 4.2 回滚机制

**自动回滚触发条件**：
- 错误率 > 5%（5 分钟窗口）
- P99 延迟 > 基线×2
- 健康检查连续 3 次失败
- Critical 级别告警

**LKG（Last Known Good）版本**：每次成功部署后标记为 LKG，回滚时优先选择。

### 4.3 部署审计

每次部署必须记录：
- 部署时间、执行者（Agent ID）、版本号
- 部署前/后指标对比
- 回滚计划
- 受影响的用户/服务

## 第 5 章：发布管理

### 5.1 发布 Readiness 检查

| 检查项 | 说明 |
|--------|------|
| E2E 测试 100% 通过 | 关键用户旅程覆盖 |
| Beta 用户满意度达标 | 真实用户反馈 |
| GRTR/ADCP 全通过 | 跨部门评审 |
| 性能预算合规 | 对照预算逐项验证 |
| 安全扫描无高危 | DAST + SAST |
| 用户文档完整 | 帮助文档、API 文档、Release Notes |

### 5.2 版本号规范

```
{major}.{minor}.{patch}-{autocoding}-{commit_sha_short}
```

AI 生成的 PR 不得自行决定 major/minor 版本变更，必须由人类在 DCP 中确定。

### 5.3 发布 Cadence

| 类型 | 频率 | 变更范围 |
|------|------|---------|
| Hotfix | 随时 | 仅修复 Critical Bug |
| Patch | 每周 | Bug 修复 + 小改进 |
| Minor | 每 2-4 周 | 新功能 |
| Major | 按需 | 不兼容变更 |
