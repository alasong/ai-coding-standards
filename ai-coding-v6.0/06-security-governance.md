## 第 1 章：安全底线

### 1.1 P5/P10/P17/P19 不可裁剪

任何 Process Profile（S/M/L/XL）必须遵守：

| 原则 | 要求 | 检测 |
|------|------|------|
| **P5 密钥不入代码** | 环境变量或密钥管理 | pre-commit gitleaks + CI SAST |
| **P10 数据分级** | Restricted 数据拦截 | pre-send 扫描 |
| **P17 输入校验** | 类型校验 + 边界检查 | 模糊测试 + 400 返回 |
| **P19 认证门禁** | 写端点必须认证 | 集成测试 401/403 |

### 1.2 Sub-Agent 安全约束

- Sub-Agent 继承主 Agent 的权限限制（deny 规则）
- 不得修改安全敏感文件（secrets/、.env）
- 安全相关变更必须在主 Agent 上下文中审查
- Agent Manifest 中声明 `tool_permissions.denied` 约束

## 第 2 章：企业部署安全

### 2.1 权限模型

| 角色 | 权限 | 限制 |
|------|------|------|
| L1 Agent | 辅助编码 | 每一步人工确认 |
| L2 Agent | 半自主编码 | PR 合并前人工确认 |
| L3 Agent | 受限自主 | DCP 门禁前自主 |
| L4 Agent | 完全自主 | 仅 DCP + 定期审计 |

### 2.2 审计要求

| 要求 | 说明 |
|------|------|
| **操作审计** | 所有 Agent 操作记录到 `.gate/` 目录 |
| **变更审计** | 每个 PR 必须关联 Spec 路径 |
| **合规审计** | 定期（周/月）审查 AI 代码合规率 |
| **证据链审计** | P11 要求每个声明 ≥ 2 条独立证据 |

### 2.3 MCP 安全

- MCP 服务器必须经过安全审查后方可启用
- MCP 工具调用记录必须纳入审计日志
- 外部 MCP 服务器不得访问 Restricted 数据

## 第 3 章：依赖治理

### 3.1 P24 标准库优先

引入新依赖前必须证明：
1. 标准库无法实现
2. 已有依赖无法复用
3. 不是仅因"方便"引入

### 3.2 依赖审计

| 检查项 | 工具 | 频率 |
|--------|------|------|
| 已知漏洞 | trivy / npm audit / safety | CI L3 |
| 许可证合规 | license-checker | CI L3 |
| 版本锁定 | lock 文件验证 | CI L1 |
| 未使用依赖 | depcheck | 每周 |

### 3.3 依赖更新策略

- 安全补丁：48 小时内更新
- 次版本更新：季度更新
- 主版本更新：按需评估，必须有迁移计划
- 更新后必须全量测试通过

### 3.4 供应链安全

**AI 不得自由添加新依赖。** 每次引入第三方库必须：

1. **DRF（Dependency Request Form）**：AI 编写依赖引入申请到 `.gate/dep-requests/`，包含包名、版本、用途、标准库替代可行性、许可证、传递依赖数、维护活跃度、已知漏洞、移除计划
2. **自动审批检查**（CI 执行）：标准库替代可行性、许可证兼容性、已知漏洞检查、包大小与传递依赖数、维护活跃度
3. **人工审批**（自动通过后）：架构评审 + 供应链风险评估
4. **白名单机制**：只有白名单中的直接依赖才可被 AI 引入

**CVE 响应**：
- CRITICAL：48 小时内修复或移除
- HIGH：1 周内修复
- MEDIUM：下次版本更新时修复

**SBOM（Software Bill of Materials）**：每次发布必须生成完整依赖清单，包含直接依赖 + 传递依赖的精确版本 + 许可证。

## 第 4 章：混沌工程与安全测试

### 4.1 DAST（动态安全测试）

L4+ 执行，覆盖：
- SQL 注入检测
- XSS 检测
- 越权访问检测
- 文件上传漏洞
- CSRF 检测

### 4.2 混沌注入

L5+ 生产环境混沌测试（受控条件）：
- 网络延迟注入
- 服务实例终止
- 数据库连接池耗尽
- DNS 故障

混沌测试必须在预发环境中先验证通过。

### 4.3 安全 Gate 检查项

| Gate | 检查内容 | 证据来源 |
|------|---------|---------|
| 密钥 Gate | 代码/配置/提交中无密钥 | gitleaks + SAST |
| 注入 Gate | 无 SQL/命令/代码注入 | SAST + DAST |
| 权限 Gate | 写端点有认证和角色校验 | 集成测试 |
| 数据 Gate | Restricted 数据未发送 | pre-send 扫描 |
| 依赖 Gate | 无已知高危漏洞 | trivy / audit |
