## 第 1 章：可观测性

### 1.1 三支柱

| 支柱 | 要求 | 工具 |
|------|------|------|
| **日志** | 结构化日志（JSON），含 request_id、tenant_id、用户 ID | 集中式日志系统 |
| **指标** | RED 方法（Rate、Errors、Duration），每个端点暴露指标 | Prometheus/等效 |
| **追踪** | 分布式追踪，请求跨服务传播 trace_id | OpenTelemetry/等效 |

### 1.2 健康检查

每个服务必须暴露 `/health` 端点：
- `200 OK`：服务正常
- `503 Service Unavailable`：依赖不可用
- 健康检查不暴露内部细节

### 1.3 SLO 定义

| SLO | 目标 | 告警 |
|-----|------|------|
| 可用性 | 99.9%（月度） | < 99.5% |
| P99 延迟 | < 500ms | > 1s |
| 错误率 | < 0.1% | > 1% |

### 1.4 AI 代码的可观测性要求

AI 生成的代码必须：
- 在关键路径添加结构化日志
- 不得吞掉异常（P13）
- 错误消息不泄露内部细节（安全要求）

## 第 2 章：环境管理

### 2.1 环境层级

```
local → dev → staging → canary → production
```

| 环境 | 用途 | 数据 |
|------|------|------|
| local | 开发调试 | Mock 数据 |
| dev | 集成测试 | 测试数据 |
| staging | 发布前验证 | 脱敏生产数据副本 |
| canary | 小流量验证 | 生产数据 |
| production | 用户流量 | 生产数据 |

### 2.2 环境一致性

P12：开发环境通过声明式配置管理（`.mise.toml`、`Makefile`）。
- 新开发者能一键 `make setup`
- 不靠运行时重试或手动安装

### 2.3 SLA 与缓存

| 缓存类型 | 失效策略 | TTL | 注意事项 |
|---------|---------|-----|---------|
| 应用级缓存 | LRU + TTL | 按需 | 必须有清理（P16） |
| CDN 缓存 | 按 URL + 版本号 | 长 TTL | 内容不可变 |
| 数据库缓存 | 查询结果缓存 | 短 TTL | 写操作必须失效 |

缓存必须有失效策略（P16），不得创建无 TTL 的全局缓存。

## 第 3 章：数据治理

### 3.1 数据分类

| 级别 | 定义 | 处理要求 |
|------|------|---------|
| **Public** | 公开数据 | 正常传输和存储 |
| **Internal** | 内部数据 | 传输加密，访问控制 |
| **Confidential** | 敏感数据 | 端到端加密，最小化访问 |
| **Restricted** | 密钥、PII、生产 DB | 不得发送给 AI，pre-send 拦截 |

### 3.2 AI 数据分级（P10）

发送给 AI 的数据必须分类：
- Public/Internal 可以发送
- Confidential 需脱敏后发送
- Restricted 禁止发送（pre-send 扫描拦截）

### 3.3 数据保留

| 数据类型 | 保留时间 | 清理方式 |
|---------|---------|---------|
| 日志 | 30-90 天 | 自动轮转 |
| 审计日志 | 1 年+ | 归档到冷存储 |
| 用户数据 | 按法规要求 | 支持数据删除（GDPR） |

## 第 4 章：国际化与可访问性

### 4.1 i18n

- 从第一天支持 i18n（不得硬编码字符串到代码中）
- 使用 locale 文件管理翻译
- 日期/时间/数字格式使用 locale 感知的方法

### 4.2 A11y

- 语义化 HTML 结构
- 图片有 alt 文本
- 表单有 label 关联
- 键盘导航支持
- 色彩对比度符合 WCAG AA

### 4.3 响应式设计

- 移动端优先
- 最小支持宽度 320px
- 触摸目标 ≥ 44×44px
